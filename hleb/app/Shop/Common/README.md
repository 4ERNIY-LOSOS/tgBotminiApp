# Общий Модуль (`app/Shop/Common/`)

Общий модуль (`Common`) внутри `app/Shop/` служит центральным местом для общего кода, утилит, базовых классов, интерфейсов, трейтов и сервисов, которые используются несколькими другими подмодулями Магазина (например, Product, Cart, Order, Admin).

## Назначение:

Основная цель этого модуля:

*   **Способствовать Повторному Использованию Кода:** Избегать дублирования кода, предоставляя общие функциональные возможности в одном месте.
*   **Улучшить Поддерживаемость:** Облегчить обновление общей логики, так как изменения нужно будет вносить только в одном месте.
*   **Улучшить Организацию:** Поддерживать чистоту структуры модуля Магазина, отделяя компоненты общего назначения от специфической для домена логики.

## Потенциальные Компоненты:

*   **Базовые Классы:**
    *   `BaseShopController.php`: Базовый контроллер, который другие контроллеры магазина могут наследовать для получения общей функциональности, такой как проверка аутентификации пользователя, инициализация общих сервисов или подготовка общих данных для представлений.
    *   `BaseShopService.php`: (Если применимо) Базовый класс сервиса.
*   **Сервисы:**
    *   `TelegramService.php`: Сервис для инкапсуляции взаимодействий с Telegram Bot API (отправка сообщений, обработка обновлений и т.д.), используемый различными модулями, которым необходимо взаимодействовать с Telegram.
    *   `ShopLogger.php`: (Если необходим специальный логгер для действий магазина) Выделенный сервис логирования.
    *   Утилитарные сервисы для задач, таких как форматирование валюты, манипуляции с датами, специфичные для нужд магазина, и т.д.
*   **Интерфейсы и Трейты:** Общие интерфейсы или трейты, которые определяют контракты или предоставляют повторно используемое поведение для других классов в модуле Магазина.
*   **Вспомогательные Функции:** Утилитарные функции, которые широко используются в бэкенде магазина.
*   **Middleware (Посредники):** (Если не размещены в `app/Middlewares` верхнего уровня или в более специфичной директории посредников Магазина) Общие посредники для маршрутов магазина, например, `UserAuthMiddleware`, `AdminAuthMiddleware`. Первоначальная структура проекта предполагает, что они могут находиться в `hleb/app/Middlewares/` или в более специфичной `hleb/app/Shop/Common/Middlewares/`, если требуется такой уровень детализации.

Централизуя эти общие элементы, остальные подмодули Магазина могут более эффективно сосредоточиться на своих специфических обязанностях.
