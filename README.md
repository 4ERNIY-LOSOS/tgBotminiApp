# 1. Название Проекта

**Интернет-Магазин в Telegram на Hleb**

Этот проект представляет собой основу для создания интернет-магазина, интегрированного с Telegram через бота и Mini App, разработанного на PHP фреймворке Hleb и работающего в Docker-контейнерах.

# 2. Быстрый Старт / Установка и Запуск

## Предварительные Требования

*   Установленный Git.
*   Установленные Docker и Docker Compose.

## Шаг 1: Клонирование Репозитория

1.  Откройте терминал или командную строку.
2.  Выполните следующие команды:
    ```bash
    git clone <URL_РЕПОЗИТОРИЯ_ВАШЕГО_ПРОЕКТА> shop-telegram-hleb
    cd shop-telegram-hleb
    ```
    Замените `<URL_РЕПОЗИТОРИЯ_ВАШЕГО_ПРОЕКТА>` на актуальный URL вашего Git-репозитория.

## Шаг 2: Настройка Переменных Окружения (`.env`)

### Что такое `.env.example` и `.env`?
*   Файл `.env.example` — это шаблон, который показывает, какие переменные окружения необходимы для работы приложения. Он содержит список всех настроек с примерами или пустыми значениями.
*   Файл `.env` — это файл, который вы создаете (копируя `.env.example`), и в котором вы указываете ваши **реальные** значения для настроек, включая секретные ключи (например, токен вашего Telegram бота). **Этот файл (`.env`) не должен добавляться в систему контроля версий (Git)**, так как он содержит конфиденциальную информацию. В данном проекте он уже добавлен в `.gitignore`.

### Создание вашего файла `.env`:
В корневой директории проекта выполните:
```bash
cp .env.example .env
```

### Обязательные настройки в `.env`:
1.  Откройте созданный файл `.env` вашим любимым текстовым редактором.
2.  Найдите строку `TELEGRAM_BOT_TOKEN=""` и вставьте ваш уникальный токен Telegram бота, полученный от [@BotFather](https://t.me/BotFather).
    *   Пример: `TELEGRAM_BOT_TOKEN="1234567890:ABCDEFGHIJKLMNopqrstuvwxyz12345"`
3.  (Опционально) Проверьте другие настройки, такие как `SERVER_EXTERNAL_PORT` (порт, на котором будет доступен ваш веб-сервер локально), `DATABASE_EXTERNAL_PORT` (порт для подключения к БД извне контейнера), `MYSQL_USER`, `MYSQL_PASSWORD`, и измените их при необходимости для вашего локального окружения. Стандартные значения обычно подходят для первого запуска.

### Замечание по `MINI_APP_BASE_URL`:
При использовании Docker, как описано ниже, URL для Mini App (используемый для кнопки в боте) будет автоматически определен и использован приложением. Значение `MINI_APP_BASE_URL` в `.env` файле может служить резервным вариантом (если автоматическое определение не удалось) или использоваться для локальной разработки без Docker.

## Шаг 3: Сборка и Запуск Docker Контейнеров

1.  Если это первый запуск или вы вносили изменения в `Dockerfile` (или связанные скрипты в `docker/`), рекомендуется собрать образы:
    ```bash
    docker-compose build
    ```
2.  Запуск всех сервисов в фоновом режиме:
    ```bash
    docker-compose up -d
    ```
    Эта команда запустит веб-сервер Nginx, PHP-FPM, базу данных MariaDB и сервис инициализации веб-хука.

## Шаг 4: Автоматическая Настройка (Что происходит "под капотом")

После запуска Docker контейнеров сервис `webhook-init` автоматически выполнит следующие действия:
1.  Запустит `localtunnel` для создания публичного HTTPS URL, который будет указывать на ваш локальный веб-сервер (nginx), работающий внутри Docker.
2.  Зарегистрирует этот публичный URL как веб-хук для вашего Telegram бота (используя ваш `TELEGRAM_BOT_TOKEN` из `.env`). Это позволит Telegram отправлять обновления (сообщения, команды от пользователей) на ваше приложение.
3.  Сохранит этот публичный URL в специальный файл внутри Docker (`/run/shared_config/localtunnel_url.txt`). Ваше PHP-приложение (в частности, контроллер `ProductController`) сможет прочитать этот URL и использовать его для формирования корректной ссылки на Mini App в кнопке, отправляемой в ответ на команду `/start`.

## Шаг 5: Проверка Работоспособности

1.  Откройте Telegram и найдите вашего бота (имя, которое вы задали при его создании через BotFather).
2.  Отправьте команду `/start` вашему боту.
3.  В ответ вы должны получить приветственное сообщение и кнопку "🛍️ Открыть Магазин (Mini App)".
4.  Нажатие на эту кнопку должно открыть интерфейс вашего Mini App прямо в Telegram.

## Шаг 6: Просмотр Логов и Отладка

Если что-то пошло не так, или вы хотите увидеть детали работы сервисов:
*   **Логи сервиса инициализации веб-хука (`webhook-init`):**
    Здесь вы можете увидеть URL, полученный от `localtunnel`, и статус установки веб-хука Telegram.
    ```bash
    docker-compose logs webhook-init
    ```
*   **Логи PHP-приложения (`php` сервис):**
    Помогут отследить ошибки в работе вашего Hleb-приложения или увидеть логи, которые вы пишете с помощью `Log::info()` и т.п.
    ```bash
    docker-compose logs php
    ```
*   **Логи веб-сервера Nginx (`nginx` сервис):**
    Содержат информацию о входящих HTTP-запросах.
    ```bash
    docker-compose logs nginx
    ```

## Шаг 7: Остановка Окружения

1.  Чтобы остановить все запущенные Docker-контейнеры, связанные с проектом:
    ```bash
    docker-compose down
    ```
    Эта команда остановит и удалит контейнеры, но сохранит данные вашей БД (так как для нее настроен volume).
2.  При следующем запуске `docker-compose up -d` сервис `webhook-init` снова получит новый (обычно другой) URL от `localtunnel` и автоматически обновит веб-хук для вашего бота.

# 3. Обзор Архитектуры Проекта

Проект построен на PHP фреймворке **Hleb** и запускается в **Docker** контейнерах, управляемых через `docker-compose`.

**Ключевые компоненты:**
*   **Веб-сервер Nginx (`nginx` сервис):** Принимает все HTTP-запросы и направляет PHP-запросы на обработку PHP-FPM. Также обслуживает статические файлы Mini App.
*   **PHP-FPM (`php` сервис):** Исполняет PHP-код приложения Hleb, включая всю бизнес-логику магазина, обработку команд бота и API для Mini App.
*   **MariaDB (`db` сервис):** Система управления базами данных, используемая для хранения информации о товарах, заказах, пользователях и т.д.
*   **`webhook-init` сервис:** Вспомогательный сервис, который при старте Docker-окружения:
    *   Запускает `localtunnel` для получения публичного HTTPS URL.
    *   Автоматически регистрирует этот URL как веб-хук для Telegram бота.
    *   Делает этот URL доступным для PHP-приложения, чтобы оно могло корректно формировать ссылку на Mini App.
    Это ключевой элемент для удобной локальной разработки и тестирования веб-хуков Telegram.
*   **Telegram Бот:** Интерфейс взаимодействия с пользователем в Telegram. Обрабатывает команды (например, `/start`) и предоставляет кнопку для запуска Mini App. Основная логика обработки команд бота находится в `hleb/app/Shop/Product/Controllers/ProductController.php`.
*   **Telegram Mini App (TMA):** Веб-приложение (HTML, CSS, JS), расположенное в директории `hleb/public/shop_mini_app/`. Оно открывается внутри интерфейса Telegram и взаимодействует с бэкендом (PHP-приложением Hleb) через специально разработанные API-маршруты для получения данных о товарах, управления корзиной, оформления заказов и т.д.

Более подробную информацию о структуре каждого модуля можно найти в соответствующих `README.md` файлах внутри директорий (например, `hleb/app/Shop/README.md` (на русском), `docker/webhook-init/README.md` (на русском)).

# 4. Структура Директорий (Ключевые)

Ниже представлена основная структура директорий проекта:

*   `./docker/`: Содержит конфигурации Docker (`Dockerfile` для PHP, `nginx.conf` для Nginx) и скрипты.
    *   `webhook-init/`: Включает `Dockerfile` и `entrypoint-webhook.sh` для сервиса автоматической настройки `localtunnel` и веб-хука Telegram.
*   `./hleb/`: Корневая директория PHP-приложения на фреймворке Hleb.
    *   `app/Shop/`: Основная директория для всей бизнес-логики интернет-магазина. Разделена на подмодули (Admin, Ai, Cart, Common, Order, Product, User).
    *   `config/`: Конфигурационные файлы Hleb. Включает `main.php`, `database.php`, а также специфичные для проекта `telegram.php` и `shop.php`.
    *   `public/`: Публично доступная директория веб-сервера.
        *   `index.php`: Главный входной файл Hleb.
        *   `shop_mini_app/`: Содержит все статические файлы (HTML, CSS, JS, изображения) для Telegram Mini App.
    *   `routes/map.php`: Файл определения всех маршрутов приложения, включая маршруты для веб-страниц (если есть), API для Mini App и эндпоинт для веб-хука Telegram.
*   `.env.example`: Шаблон файла переменных окружения.
*   `.env`: Ваш персональный файл переменных окружения (создается из `.env.example`).
*   `docker-compose.yml`: Файл для определения и управления мультиконтейнерным Docker-приложением.
*   `README.md`: Данный файл.

# 5. Дальнейшая Разработка

1.  **База Данных:**
    *   Определите схему вашей базы данных для сущностей магазина: товары, категории, пользователи, заказы, позиции заказов, избранное и т.д.
    *   Создайте таблицы в базе данных. Вы можете использовать SQL-скрипты или, если Hleb или подключенная библиотека ORM поддерживает миграции, воспользуйтесь ими.
2.  **Модели Данных:**
    *   Реализуйте PHP классы моделей в соответствующих директориях `hleb/app/Shop/.../Models/` для взаимодействия с таблицами вашей базы данных.
3.  **Модули Магазина:**
    *   Последовательно развивайте функционал в каждом подмодуле директории `hleb/app/Shop/`:
        *   **Product:** Отображение каталога, карточек товаров, поиск, фильтрация.
        *   **Cart:** Логика добавления в корзину, просмотра, изменения, удаления товаров из корзины.
        *   **Order:** Процесс оформления заказа, сохранение заказов, интеграция с платежными системами (если планируется).
        *   **User:** Регистрация/аутентификация (если требуется дополнительная к Telegram ID), профиль пользователя, история заказов, управление избранным.
        *   **Admin:** Интерфейс для администратора для управления товарами, категориями, заказами, пользователями.
        *   **Ai:** (Если планируется) Интеграция с ИИ для чат-бота, рекомендаций и т.д.
4.  **Telegram Mini App (TMA) UI/UX:**
    *   Разрабатывайте пользовательский интерфейс и клиентскую логику Mini App в директории `hleb/public/shop_mini_app/` (HTML, CSS, JavaScript).
    *   Обеспечьте взаимодействие TMA с бэкендом через API-маршруты.
5.  **Команды Telegram Бота:**
    *   Расширяйте набор команд, обрабатываемых вашим ботом. Логика обработки команд находится в `hleb/app/Shop/Product/Controllers/ProductController.php` (метод `handleWebhook`). При необходимости можно вынести эту логику в отдельный, более специализированный контроллер для бота.

# 6. Зависимости PHP

*   Проект использует **Composer** для управления PHP-зависимостями. Основные зависимости, такие как фреймворк Hleb и Telegram Bot SDK, указаны в файле `hleb/composer.json`.
*   При первоначальной сборке Docker-образа (`docker-compose build`) зависимости Composer должны устанавливаться автоматически согласно `composer.lock` (если он есть и актуален) или `composer.json`.
*   Если вам необходимо обновить зависимости или добавить новые вручную внутри уже работающего `php` контейнера, выполните:
    ```bash
    docker-compose exec php composer update
    ```
    Или для установки новой зависимости:
    ```bash
    docker-compose exec php composer require vendor/package
    ```
    После таких изменений может потребоваться пересборка образа (`docker-compose build php`), чтобы изменения сохранились для последующих запусков.
